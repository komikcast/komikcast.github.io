35. Jika permission yang dimiliki oleh sebuah file adalah –rwxr-xrw-. Tuliskanlah perintah untuk merubah permission file tersebut menjadi sebagai berikut: User memiliki permission membaca dan menulis, Group memiliki permission terbuka semua, sedangkan Others memiliki permission membaca dan menjalankan!


Perintah untuk mengubah permission file tersebut sehingga sesuai dengan soal adalah chmod 675 [nama file].

Pembahasan

Sebelum mencoba untuk mengubah permission file di sistem operasi Linux/Unix melalui terminal, terlebih dahulu harus diketahui struktur dan arti dari kode permission file yang ada di Linux/Unix. Untuk mengetahui permission file dan folder yang ada di linux, eksekusi perintah ls -l pada terminal (lihat gambar 1). Pada gambar tersebut didapat 2 hasil sebagai berikut

-rwxr-xrw- 1 ubuntu ubuntu   0 Jan 22 19.05 CONTOHFILE.txtdrwxrwxr-x 2 ubuntu ubuntu 4096 Jan 22 19.16 CONTOHFOLDER

Kode -rwxr-xrw- dan drwxrwxr-x pada hasil diatas merupakan contoh dari kode permission file. Tanda "-" di awal adalah penanda file, sedangkan tanda "d" diawal merupakan penanda folder/direktori. Setelah tanda "-" dan "d" diawal kode, terdapat  representasi hak akses yang terdiri atas set bilangan biner sebanyak 9 bit, terdiri atas 3 hak akses sebagai berikut :

3 bit pertama merupakan hak akses yang diberikan pada user (pemilik file). Perhatikan bahwa file CONTOHFILE.txt memiliki kode rwx di 3 bit pertama. Kode rwx menandakan bahwa user dapat membaca (r artinya read), menulis ulang(w artinya write), dan dapat mengeksekusi /menjalankan file (x artinya execute). Representasi dari kode rwx dalam bentuk biner adalah 111(r, w, x bernilai 1, sedangkan tanda "-" bernilai 0) sedangkan bentuk desimalnya adalah (1 x 2²) + (1 x 2¹) + (1 x 2⁰) = 7 .3 bit setelahnya menandakan hak akses untuk anggota group. Kode r-x pada permission file CONTOHFILE.txt memiliki arti bahwa anggota group dapat membaca file dan mengeksekusi file, namun tidak dapat menulis ulang file (write tidak diperbolehkan, ditandai dengan tanda "-"). Representasi lain dari kode r-x adalah 101(bentuk biner) atau 5(bentuk desimal).3 bit terakhir menandakan hak akses untuk other/pengguna lain yang bukan pemilik maupun anggota group. kode rw- pada permission file CONTOHFILE.txt memiliki arti pengguna lain dapat membaca dan menulis ulang file, tetapi tidak dapat mengeksekusinya (execute tidak diperbolehkan, ditandai dengan tanda "-"). Representasi lain dari kode rw- adalah 110(bentuk biner) atau 6(bentuk desimal).

Setelah memahami struktur dan arti dari kode permission file dan folder di Linux/Unix, maka selanjutnya tinggal mengubah permission file dengan perintah chmod. Bentuk dasar dari perintah chmod adalah

chmod [kode permission] [nama file/folder]

Perhatikan permission yang diminta soal sebagai berikut :

User memiliki permission membaca(r) dan menulis(w), kodenya rw-. Bentuk binernya adalah 110 dan bentuk desimalnya adalah (1 × 2²) + (1 × 2¹) + (0 × 2⁰) = 6.Group memiliki permission terbuka semua, kodenya rwx. Bentuk binernya adalah 111 dan bentuk desimalnya adalah (1 × 2²) + (1 × 2¹) + (1 × 2⁰) = 7.Other memiliki permission membaca(r) dan menjalankan(x), kodenya r-x. Bentuk binernya adalah 101 dan bentuk desimalnya adalah (1 × 2²) + (0 × 2¹) + (1 × 2⁰) = 5.

Maka, jika digabungkan kode permission file yang sesuai adalah rw-rwxr-x atau bentuk desimalnya adalah 675. Maka perintah untuk mengubah permission dari file CONTOHFILE.txt sesuai dengan permintaan soal dapat ditulis

chmod 675 CONTOHFILE.txt

(catatan : perintah diatas sebagai contoh saja)

Setelah perintah diatas dieksekusi, maka permission-nya juga berubah. Eksekusi perintah beserta hasilnya dapat dilihat pada lampiran gambar nomor 2.

Jadi, perintah yang dimaksud dalam soal adalah chmod 675 [nama file].
